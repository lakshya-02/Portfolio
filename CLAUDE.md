# Current Development Scope

This is the first version of the portfolio.

Do NOT spend time creating real content.

Treat all content as placeholders.

Use dummy text, placeholder project names, placeholder descriptions, and sample timelines.

The goal is to build a beautiful, reusable portfolio template that can be populated later.

Actual content will be added manually or generated later using AI.

---

# Theme

The website should have a bold, modern, futuristic aesthetic.

Primary color palette:

- Pure Black (#000000)
- Dark Gray
- Light Gray
- White
- Hot Pink / Neon Pink as the primary accent
- Small amounts of Electric Blue as a secondary accent

The overall feeling should be:

- Premium
- Minimal
- Modern
- Confident
- Clean

Do not use colorful gradients everywhere.

Accents should be used sparingly to draw attention.

Black should dominate the interface.

---

# Theme Switching

Do NOT implement light mode.

Do NOT implement theme toggling.

The website should only support a dark theme.

Everything should be designed around a black-first experience.

---

# Typography

Choose the best font combination automatically.

Prioritize readability and aesthetics over trendy fonts.

Large bold headings.

Comfortable body text.

Excellent spacing.

Avoid decorative fonts.

---

# Images

Use as few images as possible.

The design should rely primarily on:

- Typography
- Layout
- Motion
- Spacing
- Shapes
- Cards
- Icons

Placeholder illustrations or abstract graphics are acceptable if needed.

Avoid depending on large hero images.

---

# Content

Everything can be placeholder content.

Examples:

Example timeline

No real portfolio data is required.

The structure matters more than the content.

---

# Responsiveness

The portfolio must work beautifully on:

- Desktop
- Laptop
- Tablet
- Mobile

Design mobile-first where appropriate.

Layouts should adapt naturally without breaking.

Avoid horizontal scrolling.

Maintain consistent spacing across all screen sizes.

---

# UI Style

The interface should feel:

Bold

Elegant

Minimal

Aesthetic

Premium

Avoid visual clutter.

Every section should have breathing room.

Prefer clean layouts over complex ones.

---

# Components

Keep components reusable.

Avoid unnecessary complexity.

Use reusable cards, buttons, badges, and section layouts that can easily be populated with real data later.

---

# Animations

Animations should be subtle and meaningful.

Use motion to improve user experience, not to distract.

Simple fades.

Smooth transitions.

Hover interactions.

Small movement.

No excessive animations.

---

# Data

Keep all displayed content separate from the UI.

Even placeholder data should come from structured data files.

Do not hardcode text directly inside components.

---

# Overall Goal

Think of this project as creating a premium portfolio template rather than a finished personal website.

The focus is on creating a reusable, elegant, responsive foundation that can later be filled with real content.

The final result should feel clean, bold, modern, and memorable while remaining simple and easy to maintain.
